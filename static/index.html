Why Customer Support?

Customer support is a high-volume, real-world task that simultaneously demands multi-step tool use, policy reasoning, structured decision-making, and clear communication — making it an ideal RL benchmark. Unlike toy problems, every component of the reward signal reflects genuine business value: resolved tickets, accurate policy application, and satisfied customers.

Architecture
CustomerSupportEnvironment  (OpenEnv-compatible)
├── reset(task_name)  →  SupportObservation
├── step(action)      →  SupportObservation + reward
└── state             →  SupportState

Tools available to the agent:
  lookup_order · lookup_account · lookup_billing
  lookup_knowledge_base · submit_resolution · escalate

Reward: dense — every step, not just at termination
Grader: deterministic 0.0–1.0 score per task
Extending this environment

Add more tasks in server/tasks.py · Expand fixtures in server/knowledge_base.py · Replace keyword search with vector embeddings for the KB · Implement GRPO training loop using TRL and this env's reward signal
